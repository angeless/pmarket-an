## 第一章：跨平台套利策略 (Cross-Platform Arbitrage)

### 1.1 套利原理

**市场低效性来源:**
- Kalshi 用户基数小（51K周活）
- Polymarket 用户基数大（流动性高）
- 信息传播延迟：Kalshi 比 Polymarket 慢 5-30 分钟
- 结算规则差异：同一事件可能有不同 deadline

**套利公式:**
```
套利利润 = |Price_Kalshi - Price_Polymarket| - 双边手续费

触发条件:
价差 > 3% (保守) 或 > 2% (激进)
```

### 1.2 实操策略

#### 策略 A: 同事件价差套利

**适用市场:**
- 同时在 Kalshi 和 Polymarket 上架的事件
- 高关注度：选举、FOMC、重大体育事件

**操作步骤:**
1. 监控同一事件的实时价格
2. 当价差 > 3% 时触发
3. 在低价平台买入，高价平台卖出
4. 等待价格收敛或结算

**示例:**
```
事件: "Trump wins 2026 midterms"
Kalshi YES: 0.58
Polymarket YES: 0.52
价差: 0.06 (11.5%)

操作:
- Polymarket 买 YES @ 0.52 ($1000)
- Kalshi 卖 YES @ 0.58 ($1000)
- 净收益: ~$60 (扣除手续费)
```

**风险:**
- 价格可能继续 diverge（ divergence 扩大）
- 结算规则不同（注意 deadline）
- 资金两边锁定（需要双倍本金）

#### 策略 B: 信息延迟套利

**原理:**
- 重大新闻/数据发布时
- Polymarket 反应快（秒级）
- Kalshi 反应慢（分钟级）

**操作:**
1. 监控新闻源（Twitter/X, 官方发布）
2. Polymarket 价格已变动时
3. 立即在 Kalshi 同向操作
4. Kalshi 价格追上后平仓

**示例:**
```
FOMC 宣布降息
时间: 14:00 EST

14:00:00 - Polymarket 价格跳涨
14:02:00 - Kalshi 价格未变
14:02:30 - 在 Kalshi 买入
14:05:00 - Kalshi 价格追上
14:05:30 - 平仓获利
```

**关键:**
- 需要低延迟新闻源
- 动作要快（2-5分钟窗口）
- 高波动性 = 高风险

### 1.3 套利监控工具

**需要开发:**
```python
# 跨平台价格监控
class CrossPlatformMonitor:
    def __init__(self):
        self.threshold = 0.03  # 3% 触发线
        
    def check_arbitrage(self, market_kalshi, market_polymarket):
        diff = abs(market_kalshi.price - market_polymarket.price)
        if diff > self.threshold:
            return {
                'opportunity': True,
                'buy_platform': 'Kalshi' if market_kalshi.price < market_polymarket.price else 'Polymarket',
                'sell_platform': 'Polymarket' if market_kalshi.price < market_polymarket.price else 'Kalshi',
                'expected_profit': diff - 0.015  # 扣除手续费
            }
```

**监控频率:**
- 高流动性市场：每 30 秒
- 中流动性市场：每 5 分钟
- 低流动性市场：每小时

### 1.4 手续费与成本分析

| 平台 | 交易费 | 入金费 | 出金费 | 资金成本 |
|------|--------|--------|--------|---------|
| Kalshi | 0.5% | 免费(ACH) | 免费(ACH) | 美元 |
| Polymarket | 0.5-2% | USDC网络费 | USDC网络费 | USDC |

**总成本:**
- 双边交易费：~1-2.5%
- 滑点（大额）：0.5-1%
- 资金占用成本：年化 5%（机会成本）

**盈亏平衡点:**
- 价差 > 3.5% (保守)
- 价差 > 2.5% (激进)

## 第二章：Kalshi 低效市场捡漏指南

### 2.1 低效市场特征

**识别标准:**
- 日交易量 < $100,000
- 买卖价差 > 2%
- 价格长时间不更新
- 订单簿深度浅

**典型低效市场:**
- 小众体育项目（非NBA/NFL）
- 地方选举（州长、市长）
- 冷门经济指标
- 新奇事件（"会不会下雪"等）

### 2.2 捡漏策略

#### 策略 A: 专业知识套利

**原理:**
- 你有某领域的专业知识
- 市场定价基于大众认知（往往错误）
- 利用信息差获取超额收益

**示例:**
```
市场: "当地市长选举"
你: 住在该城市，了解真实民意
市场定价: 候选人A 60%
你的判断: 候选人A 30%（基于实地观察）

操作: 买 NO (候选人A输) @ 0.40
预期收益: 150% (如果正确)
```

**关键:**
- 必须有真实信息优势
- 不能是公开信息
- 需要严格的风控（可能错）

#### 策略 B: 流动性提供

**原理:**
- 低效市场买卖价差大
- 做 Maker（挂单）可以赚取价差
- Kalshi Maker 费用 0%

**操作:**
1. 找到价差 > 3% 的市场
2. 挂买单（比最低价高 0.01）
3. 挂卖单（比最高价低 0.01）
4. 等待成交，赚取中间差价

**示例:**
```
市场: "某冷门体育比赛"
买一: 0.45
卖一: 0.49
价差: 0.04 (8.9%)

操作:
- 挂买单 @ 0.46
- 挂卖单 @ 0.48
- 如果两边都成交：赚 0.02 (4.3%)
```

**风险:**
- 可能只成交一边（单向暴露）
- 资金占用时间长
- 事件结果不确定

#### 策略 C: 时间价值套利

**原理:**
- 长期市场（几个月后结算）
- 时间价值被低估
- 买入后持有，赚取时间衰减

**适用:**
- "2026年选举"类市场
- 目前定价 0.50 左右
- 事件确定性高

**操作:**
1. 找到定价 0.45-0.55 的长期市场
2. 判断事件概率 > 60%
3. 买入并持有到临近结算
4. 价格趋向 0.60-0.80

**示例:**
```
市场: "2026年中期选举共和党赢众议院"
当前价格: 0.52
你的判断: 历史规律 + 当前民调 = 70%

操作: 买 YES @ 0.52
持有: 6个月
预期结算: 0.70
收益: 34.6%
```

### 2.3 低效市场筛选工具

**筛选条件:**
```python
def find_inefficient_markets(markets):
    opportunities = []
    for m in markets:
        if m.volume_24h < 100000 and \
           m.spread > 0.02 and \
           m.days_to_resolution > 30:
            opportunities.append(m)
    return sorted(opportunities, key=lambda x: x.spread, reverse=True)
```

## 第三章：Polymarket 情绪套利量化方法

### 3.1 情绪套利原理

**核心公式:**
```
市场定价 = 真实概率 + 情绪偏差

套利机会:
情绪偏差 > 10% 时，逆向操作
情绪偏差 < -10% 时，正向操作
```

**情绪来源:**
- Twitter/X 热搜
- Reddit 讨论热度
- 新闻媒体 tone
- Google Trends

### 3.2 情绪量化指标

#### 指标 1: 社交媒体情绪指数

**数据来源:**
- Twitter API（推文情绪分析）
- Reddit 评论情感分析
- 新闻标题关键词提取

**计算方法:**
```python
def calculate_sentiment_score(keyword, timeframe='1h'):
    tweets = fetch_tweets(keyword, timeframe)
    
    positive = sum(1 for t in tweets if analyze_sentiment(t) > 0.6)
    negative = sum(1 for t in tweets if analyze_sentiment(t) < 0.4)
    neutral = len(tweets) - positive - negative
    
    # 标准化到 -1 到 +1
    score = (positive - negative) / len(tweets)
    return score
```

**阈值设定:**
- > +0.6: 过度乐观（卖）
- < -0.6: 过度悲观（买）
- -0.3 ~ +0.3: 中性（观望）

#### 指标 2: 搜索趋势偏离

**原理:**
- Google Trends 反映大众关注度
- 关注度 ≠ 真实概率
- 高关注度往往伴随过度定价

**操作:**
1. 监控事件关键词搜索量
2. 搜索量暴增时，警惕价格泡沫
3. 搜索量低迷时，寻找低估机会

**示例:**
```
事件: "某明星离婚"
Day 1: 新闻爆出，搜索量 +500%，价格 0.30 → 0.70 (过度反应)
操作: 卖 YES @ 0.70
Day 3: 搜索量回落，价格 0.70 → 0.50
盈利: 28.6%
```

#### 指标 3: 鲸鱼资金流向

**数据来源:**
- 链上大额转账监控
- 持仓集中度变化
- 聪明钱地址跟踪

**原理:**
- 鲸鱼往往有信息优势
- 但鲸鱼也可能操纵市场
- 需要结合其他指标判断

**操作:**
1. 监控 Top 100 地址持仓变动
2. 大额买入 + 情绪正面 = 跟随
3. 大额买入 + 情绪负面 = 警惕（可能诱多）

```python
def generate_signal(market, sentiment_data):
    price = market.current_price
    sentiment = sentiment_data.score
    
    # 情绪-价格偏离度
    deviation = sentiment - (price - 0.5) * 2  # 标准化
    
    if deviation > 0.2:  # 情绪 > 价格
        return {
            'action': 'BUY_YES',
            'confidence': deviation,
            'reason': '情绪过度悲观，价格低估'
        }
    elif deviation < -0.2:  # 情绪 < 价格
        return {
            'action': 'BUY_NO',
            'confidence': -deviation,
            'reason': '情绪过度乐观，价格高估'
        }
    else:
        return {'action': 'HOLD'}
```
